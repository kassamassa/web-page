# DESIGN_SYSTEM.md
## 職業図鑑：異能者編 — パロディ設計システム

**Purpose:** 職業を指定するだけで、コンテンツ設計の全要素を導き出すための知識体系。

---

## ワークフロー

```
① 職業を指定
② このドキュメントをGeminiに渡す
③ GeminiがLayer1〜3を埋める
④ ClaudeがJSONと実装指示に変換
⑤ Claude CodeがGitHubにpush
⑥ 撮影・投稿
```

---

## Layer 1：職業分析の型

どんな職業でも以下の5軸を抽出する。

### 1-A：核心業務
その職業が「一番本質的にやっていること」を1行で。
```
フォーマット：[動詞]＋[対象]＋[目的]
例：知識を→人の記憶に→永久に刻み込む（先生）
例：外見を→他者の意思通りに→完全に変容させる（美容師）
```

### 1-B：隠れた過酷さ
一般人が知らない「しんどい部分」を3つ。
```
・繰り返しの苦痛（同じことを何度もやる）
・見えない労働（裏側でやっていること）
・感情労働（気持ちを抑えてやること）
```

### 1-C：特殊スキル
その職業だけが持つ能力を6つ（S2のスキルになる）。
```
・肉体的スキル（手先・体力など）
・認知的スキル（判断・記憶など）
・対人的スキル（読む・合わせるなど）
・時間的スキル（速さ・効率など）
・感覚的スキル（目・耳・感触など）
・精神的スキル（忍耐・集中など）
```

### 1-D：象徴的な道具・空間
その職業を代表する物と場所。
```
道具：〇〇（S0のビジュアル要素・スタンド武器になる）
空間：〇〇（S5の背景になる）
```

### 1-E：一日の感情曲線
朝〜深夜の感情の流れを6段階で（S3のタイムラインになる）。
```
時刻：感情状態 → 出来事
例：
06:30：希望 → 準備
09:00：集中 → 本番開始
12:00：疲弊 → 偽りの休憩
15:00：回復 → 後半戦
18:00：達成 → 終了
23:00：準備 → 明日への覚悟
```

## Layer 2：パロディ選定の型

職業分析の結果と以下の照合基準でアニメを選ぶ。

### 照合基準（4軸）

**A：核心業務との一致**
```
その職業の「本質動作」とアニメの「核心概念」が重なるか。
例：「知識を刻む」×「スキルを習得する」→ 転スラ ◎
例：「外見を変える」×「スタンドで変容する」→ ジョジョ ◎
```

**B：隠れた過酷さとの共鳴**
```
その職業の「しんどさ」とアニメのテーマが共鳴するか。
例：「同じ質問に何千回」×「同じ敵と何度でも戦う」→ 進撃 ◎
例：「立ちっぱなし集中」×「極限の精神力」→ ジョジョ ◎
```

**C：道具の変換しやすさ**
```
その職業の道具がアニメの武器・能力に自然に変換できるか。
例：「チョーク」→「魔法の杖」→ 転スラ ◎
例：「ハサミ」→「スタンド武器」→ ジョジョ ◎
```

**D：ビジュアルの相性**
```
そのアニメの視覚的特徴がWebサイトで再現できるか。
例：転スラ→青い光・粒子・ステータス画面 → Canvas実装可 ◎
例：ジョジョ→集中線・コマ割り・ドドドド → CSS+Canvas実装可 ◎
```

## Layer 3：実装変換の型

アニメが決まったら、以下の変換表で全要素を生成する。

### 変換マッピング

```
職業分析の要素　　　→　サイトの要素
────────────────────────────────────
核心業務　　　　　　→　S1キャッチコピーの骨格
特殊スキル×6　　　→　S2スキル名6つ
一日の感情曲線×6　→　S3タイムライン6行
隠れた過酷さ×3　　→　S4進化条件（3〜4つ）
象徴的な道具　　　　→　S0ビジュアル要素・スタンド武器
象徴的な空間　　　　→　S5背景テーマ
```

### S1キャッチコピーの型

```
フォーマット：
「本日、[職業]が[アニメ固有の動詞]しました。」

例：
転スラ → 「本日、教師に転生しました。」
ジョジョ → 「本日、美容師が覚醒しました。」
デスノート → 「本日、営業職が[裁きを下す側に]なりました。」
エヴァ → 「本日、コンビニ店員が[使徒と戦う]ことになりました。」
```

### S2スキル名の型

```
フォーマット：
タグ：[アニメ固有の能力分類] [番号]
名前：[職業スキル]を[アニメ用語]で表現した2〜4文字
読み：ひらがな
説明：1行以内、職業の具体的な業務を誇張して表現

例（転スラ）：
タグ：ユニークスキル Ⅰ
名前：無限忍耐
読み：むげんにんたい
説明：1,247回目の同じ質問にも笑顔で答える。
```

### S3タイムラインの型

```
フォーマット：
時刻：[アニメ用語で表現した出来事名]
説明：1行以内、その時間帯の業務を誇張

ルール：
・時刻は実際の業務時間に準拠
・出来事名はアニメ用語に変換
・説明は「〜する。」で終わる
・笑いどころを1〜2行に入れる
```

### S4進化条件の型

```
フォーマット：
見出し：2〜4文字の漢字（強く・短く）
説明：1行以内「〜する。」または「〜ない。」

ルール：
・4つ全て職業の「精神的な資質」を表す
・最後の1つは必ず「見えない報酬」系にする
・笑いどころを1つ入れる
```

### REVEALの型

```
フォーマット：
「要するに：[職業名]です。」

締めのvow：
「[職業の複数形]たちは、毎日[アニメ固有の動詞]している。」

例：
「先生たちは、毎日転生している。」
「美容師たちは、毎日覚醒している。」
```

## Geminiへの指示テンプレート

以下をコピーしてGeminiに渡す。

```
以下のDESIGN_SYSTEMに基づいて、
[職業名]×[アニメ名（任意）]のコンテンツ設計を行ってください。

アニメは指定しない場合、Layer2の照合基準で最適なものを選んでください。

出力形式：
1. Layer1：職業分析（5軸すべて埋める）
2. Layer2：選定したアニメとその理由（4軸で評価）
3. Layer3：変換結果
   - S1キャッチコピー
   - S2スキル名6つ（タグ・名前・読み・説明）
   - S3タイムライン6行（時刻・見出し・説明）
   - S4進化条件4つ（見出し・説明）
   - REVEALのvow

SUCCESSフレームワーク（Simple・Unexpected・Concrete・Credible・Emotional・Story）
の観点で各要素を評価し、スコアが低い場合は改善案も提示してください。

[ここにDESIGN_SYSTEM.mdの内容を貼り付ける]
```
