# DESIGN_SYSTEM.md
## 職業図鑑：異能者編 — パロディ設計システム

**Purpose:** 職業を指定するだけで、コンテンツ設計の全要素を導き出すための知識体系。

---

## ワークフロー

```
① 職業を指定
② このドキュメントをGeminiに渡す
③ GeminiがLayer1〜3を埋める
④ ClaudeがJSONと実装指示に変換
⑤ Claude CodeがGitHubにpush
⑥ 撮影・投稿
```

---

## Layer 1：職業分析の型

どんな職業でも以下の5軸を抽出する。

### 1-A：核心業務
その職業が「一番本質的にやっていること」を1行で。
```
フォーマット：[動詞]＋[対象]＋[目的]
例：知識を→人の記憶に→永久に刻み込む（先生）
例：外見を→他者の意思通りに→完全に変容させる（美容師）
```

### 1-B：隠れた過酷さ
一般人が知らない「しんどい部分」を3つ。
```
・繰り返しの苦痛（同じことを何度もやる）
・見えない労働（裏側でやっていること）
・感情労働（気持ちを抑えてやること）
```

### 1-C：特殊スキル
その職業だけが持つ能力を6つ（S2のスキルになる）。
```
・肉体的スキル（手先・体力など）
・認知的スキル（判断・記憶など）
・対人的スキル（読む・合わせるなど）
・時間的スキル（速さ・効率など）
・感覚的スキル（目・耳・感触など）
・精神的スキル（忍耐・集中など）
```

### 1-D：象徴的な道具・空間
その職業を代表する物と場所。
```
道具：〇〇（S0のビジュアル要素・スタンド武器になる）
空間：〇〇（S5の背景になる）
```

### 1-E：一日の感情曲線
朝〜深夜の感情の流れを6段階で（S3のタイムラインになる）。
```
時刻：感情状態 → 出来事
例：
06:30：希望 → 準備
09:00：集中 → 本番開始
12:00：疲弊 → 偽りの休憩
15:00：回復 → 後半戦
18:00：達成 → 終了
23:00：準備 → 明日への覚悟
```

## Layer 2：パロディ選定の型

職業分析の結果と以下の照合基準でアニメを選ぶ。

### 照合基準（4軸）

**A：核心業務との一致**
```
その職業の「本質動作」とアニメの「核心概念」が重なるか。
例：「知識を刻む」×「スキルを習得する」→ 転スラ ◎
例：「外見を変える」×「スタンドで変容する」→ ジョジョ ◎
```

**B：隠れた過酷さとの共鳴**
```
その職業の「しんどさ」とアニメのテーマが共鳴するか。
例：「同じ質問に何千回」×「同じ敵と何度でも戦う」→ 進撃 ◎
例：「立ちっぱなし集中」×「極限の精神力」→ ジョジョ ◎
```

**C：道具の変換しやすさ**
```
その職業の道具がアニメの武器・能力に自然に変換できるか。
例：「チョーク」→「魔法の杖」→ 転スラ ◎
例：「ハサミ」→「スタンド武器」→ ジョジョ ◎
```

**D：ビジュアルの相性**
```
そのアニメの視覚的特徴がWebサイトで再現できるか。
例：転スラ→青い光・粒子・ステータス画面 → Canvas実装可 ◎
例：ジョジョ→集中線・コマ割り・ドドドド → CSS+Canvas実装可 ◎
```

## Layer 3：実装変換の型

アニメが決まったら、以下の変換表で全要素を生成する。

### 変換マッピング

```
職業分析の要素　　　→　サイトの要素
────────────────────────────────────
核心業務　　　　　　→　S1キャッチコピーの骨格
特殊スキル×6　　　→　S2スキル名6つ
一日の感情曲線×6　→　S3タイムライン6行
隠れた過酷さ×3　　→　S4進化条件（3〜4つ）
象徴的な道具　　　　→　S0ビジュアル要素・スタンド武器
象徴的な空間　　　　→　S5背景テーマ
```

### S1キャッチコピーの型

```
フォーマット：
「本日、[職業]が[アニメ固有の動詞]しました。」

例：
転スラ → 「本日、教師に転生しました。」
ジョジョ → 「本日、美容師が覚醒しました。」
デスノート → 「本日、営業職が[裁きを下す側に]なりました。」
エヴァ → 「本日、コンビニ店員が[使徒と戦う]ことになりました。」
```

### S2スキル名の型

```
フォーマット：
タグ：[アニメ固有の能力分類] [番号]
名前：[職業スキル]を[アニメ用語]で表現した2〜4文字
読み：ひらがな
説明：1行以内、職業の具体的な業務を誇張して表現

例（転スラ）：
タグ：ユニークスキル Ⅰ
名前：無限忍耐
読み：むげんにんたい
説明：1,247回目の同じ質問にも笑顔で答える。
```

### S3タイムラインの型

```
フォーマット：
時刻：[アニメ用語で表現した出来事名]
説明：1行以内、その時間帯の業務を誇張

ルール：
・時刻は実際の業務時間に準拠
・出来事名はアニメ用語に変換
・説明は「〜する。」で終わる
・笑いどころを1〜2行に入れる
```

### S4進化条件の型

```
フォーマット：
見出し：2〜4文字の漢字（強く・短く）
説明：1行以内「〜する。」または「〜ない。」

ルール：
・4つ全て職業の「精神的な資質」を表す
・最後の1つは必ず「見えない報酬」系にする
・笑いどころを1つ入れる
```

### REVEALの型

```
フォーマット：
「要するに：[職業名]です。」

締めのvow：
「[職業の複数形]たちは、毎日[アニメ固有の動詞]している。」

例：
「先生たちは、毎日転生している。」
「美容師たちは、毎日覚醒している。」
```

## Layer 4：ビジュアル設計の型

### 良い背景の2条件

```
条件①：Z軸的な動き
  → 奥から手前に迫ってくる感覚
  → スクロールに連動して「近づいてくる」
  → 平面ではなく立体的な奥行き感

条件②：パロディならではの要素
  → そのアニメを知っている人が「あ！」となる
  → 知らない人でも「なんかすごい」と感じる
  → テキスト・パーティクル・幾何学のいずれかで表現
```

---

### 4-A：Z軸実装パターン（4種類）

```
Pattern A：テキスト降下型
  → 文字が奥（小さい）から手前（大きい）に降ってくる
  → 例：転スラの転生パーティクル・ジョジョのドドドド
  → Canvas実装：fontSize を time に連動して変化

Pattern B：集中線型
  → 中心点から放射状に線が広がる
  → 「何かが迫ってくる」緊張感
  → 例：ジョジョS1の集中線
  → Canvas実装：中心座標から角度を計算して線を引く

Pattern C：パーティクル遠近型
  → 粒子が奥（小さい・暗い）から手前（大きい・明るい）に動く
  → 例：転スラS0の光の柱・ジョジョS5のオーラ
  → Canvas実装：z座標を持たせてperspective変換

Pattern D：スケール変化型
  → 要素が小さい状態から大きくなりながら画面を覆う
  → 「飲み込まれる」感覚
  → GSAPのscaleアニメーションで実装
  → 例：S0タイトルの召喚・S5REVEALの収束
```

---

### 4-B：スライド別背景の役割と推奨パターン

```
S0（タイトル）：世界観の第一印象
  役割：見た人が「このアニメだ！」と即座に認識する
  推奨：Pattern A + Pattern B の組み合わせ
  強度：最大（最も派手）
  必須要素：パロディ固有のテキスト or シンボル

S1（ミッション）：キャッチコピーを引き立てる
  役割：テキストの迫力を背景が増幅する
  推奨：Pattern B（集中線）or Pattern A（テキスト）
  強度：強（テキストより背景が目立ちすぎない）
  必須要素：コントラストの強い単色背景

S2（スキル）：情報を整理して見せる
  役割：6枚のカードを際立たせる
  推奨：グリッド系（パロディのUI風）
  強度：中（情報の邪魔をしない）
  必須要素：パロディ固有のUI要素（ステータス画面・コマ割りなど）

S3（タイムライン）：リズムを作る
  役割：6行の時系列にテンポ感を与える
  推奨：Pattern A（テキスト流れ）or ベタ塗り
  強度：中（タイムラインの読みやすさを優先）
  必須要素：パロディ固有の色（ベタ塗りでも可）

S4（進化条件）：重厚感・緊張感
  役割：キーワードの重さを背景が支える
  推奨：Pattern A（ゆっくり流れるテキスト）+ 暗い背景
  強度：弱〜中（キーワードを主役にする）
  必須要素：暗さ・パロディ固有の環境光

S5（REVEAL）：カタルシス・解放感
  役割：オチの気持ちよさを最大化する
  推奨：Pattern C（オーラ・収束）
  強度：強（REVEALの瞬間だけ最大に）
  必須要素：パロディ固有のシンボルが背景に溶け込む
```
